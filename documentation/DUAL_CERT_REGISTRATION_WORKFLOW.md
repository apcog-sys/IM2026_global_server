## 🔄 Registration Workflow Overview

```
┌─────────────────────────────────────────────────────────────────┐
│  SECURITY SERVER REGISTRATION WITH DUAL CERTIFICATES (AUTH+SIGN) │
└─────────────────────────────────────────────────────────────────┘

STEP 1: Generate CSRs (Local on Security Server)
├─ openssl genrsa -out server_auth.key 2048
├─ openssl req -new -key server_auth.key -out auth.csr
├─ openssl genrsa -out server_sign.key 2048
└─ openssl req -new -key server_sign.key -out sign.csr

STEP 2: Send CSRs to Global Server
├─ Endpoint: POST /api/security-servers/request-certificates
├─ Input: auth.csr + sign.csr
├─ Global Server calls CA service (port 9002)
└─ Output: auth.crt + sign.crt (both verified by CA)

STEP 3: Register with Global Server
├─ Endpoint: POST /api/security-servers/register
├─ Input: auth.csr + auth.crt + sign.csr + sign.crt
├─ Global Server stores in database
└─ Output: Both .crt files for local storage

STEP 4: Save Files Locally
├─ Save auth.crt for HTTPS/TLS
├─ Save sign.crt for message signing
└─ Keep metadata file for audits
```

### 2. Request Certificates (Dual CSR Submission)
```
POST /api/security-servers/request-certificates

Endpoint that accepts TWO CSRs and calls CA for verification.
Returns TWO verified .crt files from the CA.

Request:
{
  "server_id": "SECURITY_SERVER_1",
  "server_name": "Primary Security Gateway",
  "organization": "My Organization",
  "auth_csr": "-----BEGIN CERTIFICATE REQUEST-----\n...\n-----END CERTIFICATE REQUEST-----",
  "sign_csr": "-----BEGIN CERTIFICATE REQUEST-----\n...\n-----END CERTIFICATE REQUEST-----",
  "address": "10.0.0.50",
  "network_instance": "EE"
}

Response (200):
{
  "status": "success",
  "message": "Both AUTH and SIGN certificates generated by CA",
  "auth_certificate": {
    "certificate_id": "cert-uuid-1",
    "certificate_type": "auth",
    "crt_file": "-----BEGIN CERTIFICATE-----\n...",
    "thumbprint": "SHA256:...",
    "expiry_date": "2027-04-09T10:30:00Z",
    "issued_date": "2026-04-09T10:30:00Z"
  },
  "sign_certificate": {
    "certificate_id": "cert-uuid-2",
    "certificate_type": "sign",
    "crt_file": "-----BEGIN CERTIFICATE-----\n...",
    "thumbprint": "SHA256:...",
    "expiry_date": "2027-04-09T10:30:00Z",
    "issued_date": "2026-04-09T10:30:00Z"
  },

Errors:
- 400: Invalid CSR or CA rejected certificate

```

## 📊 Database Schema

### security_server_registrations Table

```sql
CREATE TABLE security_server_registrations (
    registration_id VARCHAR(100) PRIMARY KEY,
    server_id VARCHAR(100) UNIQUE NOT NULL,
    server_name VARCHAR(255) NOT NULL,
    organization VARCHAR(255) NOT NULL,
    address VARCHAR(100) NOT NULL,
    port INT NOT NULL,
    network_instance VARCHAR(50) DEFAULT 'EE',
    
    -- AUTH Certificate Fields
    auth_csr LONGTEXT NOT NULL,           -- Original CSR
    auth_crt LONGTEXT NOT NULL,           -- Signed CRT from CA
    auth_certificate_id VARCHAR(100),     -- CA-assigned ID
    auth_thumbprint VARCHAR(512),         -- SHA256 hash
    auth_expiry_date TIMESTAMP,           -- Expiration date
    
    -- SIGN Certificate Fields  
    sign_csr LONGTEXT NOT NULL,           -- Original CSR
    sign_crt LONGTEXT NOT NULL,           -- Signed CRT from CA
    sign_certificate_id VARCHAR(100),     -- CA-assigned ID
    sign_thumbprint VARCHAR(512),         -- SHA256 hash
    sign_expiry_date TIMESTAMP,           -- Expiration date
    
    -- Metadata
    status VARCHAR(50) DEFAULT 'registered',
    created_by VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## ✅ Files Returned to Security Server

### After Registration (Endpoint 3), Security Server receives:

1. **`{SERVER_ID}_auth.crt`**
   - Contains: AUTH certificate from CA
   - Use for: HTTPS/TLS connections
   - Combine with: `server_auth.key` (kept locally)

2. **`{SERVER_ID}_sign.crt`**
   - Contains: SIGN certificate from CA
   - Use for: Digital signatures and message verification
   - Combine with: `server_sign.key` (kept locally)

3. **`{SERVER_ID}_registration.metadata`**
   - Contains: Registration details, certificate info, timestamps
   - Use for: Audit trail and records

## 🔗 Integration with CA Service

### Global Server calls CA on Port 9002

```
For AUTH Certificate:
POST http://localhost:9002/api/certificates/generate-auth
Input: auth.csr
Output: auth.crt + certificate_id + thumbprint

For SIGN Certificate:
POST http://localhost:9002/api/certificates/generate-sign
Input: sign.csr
Output: sign.crt + certificate_id + thumbprint
```

## 📝 Example Usage

### Quick Test Script
```bash
#!/bin/bash

# 1. Generate CSRs
openssl genrsa -out server_auth.key 2048
openssl req -new -key server_auth.key -out auth.csr

openssl genrsa -out server_sign.key 2048
openssl req -new -key server_sign.key -out sign.csr

AUTH_CSR=$(cat auth.csr | tr '\n' '\\n')
SIGN_CSR=$(cat sign.csr | tr '\n' '\\n')

# 2. Request certificates
CERTS=$(curl -s -X POST http://localhost:9000/api/security-servers/request-certificates \
  -H "Content-Type: application/json" \
  -d "{\"server_id\":\"SECURITY_SERVER_1\",\"server_name\":\"Primary\",\"organization\":\"Org\",\"auth_csr\":\"$AUTH_CSR\",\"sign_csr\":\"$SIGN_CSR\",\"address\":\"10.0.0.50\",\"network_instance\":\"EE\"}")

# 3. Extract data
AUTH_CRT=$(echo $CERTS | jq -r '.auth_certificate.crt_file')
SIGN_CRT=$(echo $CERTS | jq -r '.sign_certificate.crt_file')
# ... (extract other fields)

# 4. Register
curl -s -X POST http://localhost:9000/api/security-servers/register \
  -H "Content-Type: application/json" \
  -d "{\"registration_id\":\"REG_SS_001\",...}"

# 5. Check registration
curl http://localhost:9000/api/security-servers/registrations
```
