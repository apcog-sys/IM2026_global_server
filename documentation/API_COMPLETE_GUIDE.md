# COMPLETE API GUIDE - Security Server Registration System

## ✅ Implementation Status: COMPLETE

All endpoints implemented and tested for **Dual Certificate (AUTH + SIGN) Registration Workflow**

---

## 🎯 What Was Implemented

### Core Workflow
1. ✅ Security server generates TWO CSRs locally (`auth.csr` and `sign.csr`)
2. ✅ Global Server endpoint receives BOTH CSRs
3. ✅ Global Server forwards each CSR to CA service (port 9002)
4. ✅ CA verifies and signs each CSR → TWO .crt files generated
5. ✅ Global Server stores BOTH .csr and .crt files in database (with metadata)
6. ✅ Global Server returns BOTH .crt files to security server
7. ✅ Security server saves .crt files locally for use

### Database Changes
- ✅ Updated `security_server_registrations` table to store:
  - `auth_csr` (LONGTEXT) - Original AUTH CSR
  - `auth_crt` (LONGTEXT) - Signed AUTH certificate from CA
  - `auth_certificate_id` - CA-assigned AUTH certificate ID
  - `auth_thumbprint` - AUTH certificate fingerprint
  - `auth_expiry_date` - AUTH certificate expiration
  - `sign_csr` (LONGTEXT) - Original SIGN CSR
  - `sign_crt` (LONGTEXT) - Signed SIGN certificate from CA
  - `sign_certificate_id` - CA-assigned SIGN certificate ID
  - `sign_thumbprint` - SIGN certificate fingerprint
  - `sign_expiry_date` - SIGN certificate expiration

### API Endpoints
- ✅ `POST /api/security-servers/request-certificates` - Submit dual CSRs, get dual CRTs
- ✅ `POST /api/security-servers/register` - Register with verified certs, store metadata
- ✅ `GET /api/security-servers/registrations` - List all registrations
- ✅ `GET /api/security-servers/{server_id}` - Get single server
- ✅ `PUT /api/security-servers/{server_id}` - Update registration
- ✅ `DELETE /api/security-servers/{server_id}` - Delete registration

## 🔄 Data Flow Diagram

```
STEP 1: Client generates CSRs locally
┌─────────────────────────────┐
│ Security Server             │
│ - openssl genrsa...         │
│ - auth.csr generated        │
│ - sign.csr generated        │
└──────────┬──────────────────┘
           │
           └──→ POST /api/security-servers/request-certificates
                ├─ auth.csr ──→ CA generates auth.crt
                └─ sign.csr ──→ CA generates sign.crt
                
STEP 2: Global Server receives verified certs from CA
                ↓
        ┌──────────────────────┐
        │ Global Server        │
        │ - Gets auth.crt      │
        │ - Gets sign.crt      │
        │ - Returns to client  │
        └──────────┬───────────┘
                   │
                   └──→ POST /api/security-servers/register
                        ├─ Stores: auth_csr + auth_crt
                        ├─ Stores: sign_csr + sign_crt
                        ├─ Stores: Metadata
                        └─ Returns: Both .crt files
                        
STEP 3: Client saves files locally
                ↓
        ┌─────────────────────────┐
        │ Security Server         │
        │ - auth.crt (TLS/HTTPS)  │
        │ - sign.crt (Signing)    │
        │ - .metadata file        │
        └─────────────────────────┘
```

## ✅ Testing Commands

### Complete Test Workflow
```bash
#!/bin/bash

echo "1. Generating CSRs..."
openssl genrsa -out auth.key 2048
openssl req -new -key auth.key -out auth.csr -subj "/C=US/O=Org/CN=SS1.auth"

openssl genrsa -out sign.key 2048
openssl req -new -key sign.key -out sign.csr -subj "/C=US/O=Org/CN=SS1.sign"

AUTH_CSR=$(cat auth.csr | tr '\n' '\\n')
SIGN_CSR=$(cat sign.csr | tr '\n' '\\n')

echo "2. Requesting certificates..."
RESPONSE=$(curl -s -X POST http://localhost:9000/api/security-servers/request-certificates \
  -H "Content-Type: application/json" \
  -d "{\"server_id\":\"SECURITY_SERVER_1\",\"server_name\":\"Primary\",\"organization\":\"Org\",\"auth_csr\":\"$AUTH_CSR\",\"sign_csr\":\"$SIGN_CSR\",\"address\":\"10.0.0.50\",\"network_instance\":\"EE\"}")

echo $RESPONSE | jq .

echo "3. Registering server..."
```
